# 💡 电脑关机时的定时任务解决方案

## 问题
macOS的launchd定时任务只在开机时运行，如果电脑在早上7点关机，任务不会执行。

## 🎯 推荐方案（3种）

---

## 方案1：登录时自动检查 + 每30分钟检查（推荐）

### 优点
- ✅ 开机后自动补执行错过的任务
- ✅ 全天多次检查，确保不会漏掉
- ✅ 无需额外配置
- ✅ 支持补执行前几天错过的任务

### 配置方法

```bash
# 1. 复制定时任务配置
cp /Users/baoying/Desktop/AI简报/com.aiBrief.login.plist ~/Library/LaunchAgents/

# 2. 加载任务
launchctl load ~/Library/LaunchAgents/com.aiBrief.login.plist

# 3. 立即测试
python3 /Users/baoying/Desktop/AI简报/offline-solution.py
```

### 工作原理
- 登录时自动运行一次
- 之后每30分钟检查一次是否需要更新
- 如果今天还没运行过，自动执行
- 防止重复运行（使用锁文件）

## 方案2：macOS定时唤醒

### 设置自动唤醒

```bash
# 设置每天早上6:55自动唤醒（7点前5分钟）
sudo pmset repeat wake MTWRFSU 07:00:00
```

### 查看当前设置
```bash
pmset -g sched
```

### 取消自动唤醒
```bash
sudo pmset repeat cancel
```

### 优点
- ✅ 电脑会自动唤醒执行任务
- ✅ 执行完可以继续休眠

### 缺点
- ❌ 需要确保电脑接通电源或电量充足
- ❌ 笔记本电脑在合盖状态下可能不会唤醒

## 方案3：使用云服务器（最稳定）

### 免费云服务选项

#### 1. **GitHub Actions**（推荐，完全免费）

创建 `.github/workflows/daily-update.yml`:

```yaml
name: AI简报每日更新

on:
  schedule:
    - cron: '0 23 * * *'  # UTC时间23点 = 北京时间7点
  workflow_dispatch:      # 支持手动触发

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - name: 运行更新脚本
        run: |
          # 这里运行你的更新逻辑
          python update.py
```

#### 2. **Railway**（每月$5免费额度）
- 支持定时任务
- 自动部署Python脚本
- 简单易用

#### 3. **Cron-Job.org**（免费）
- 在线cron服务
- 配置简单
- 支持HTTP请求触发

### 使用GitHub Actions的步骤

1. 创建GitHub仓库
2. 添加你的脚本
3. 创建workflow文件
4. 推送到GitHub
5. 每天自动运行

## 🎯 我的建议

### 最佳实践：方案1 + 方案2 组合

```bash
# 1. 配置登录检查（主方案）
cp /Users/baoying/Desktop/AI简报/com.aiBrief.login.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.aiBrief.login.plist

# 2. （可选）设置定时唤醒（备用）
sudo pmset repeat wake MTWRFSU 07:00:00
```

这样：
- 如果电脑开机 → 登录检查会自动补执行
- 如果电脑关机但接电源 → 定时唤醒会执行
- 双重保障，基本不会漏掉
